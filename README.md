# ДЗ 4 по КПО

Реализованы микросервисы `Orders` и `Payments`. Доступны через микросервис `Gateway`.

Обработка заказов происходит при помощи `RabbitMQ`.

Также с помощью `SvelteKit` был реализован фронтенд для приложения.

## Стек

* **Backend:** Rust (Actix Web, Tokio, Lapin, SurrealDB SDK)
* **СУБД:** SurrealDB
* **Брокер сообщений:** RabbitMQ
* **Frontend:** SvelteKit + TypeScript + TailwindCSS
* **Документация:** OpenApi (Swagger) через Utoipa

## Архитектура

Проект состоит из 4-х контейнеризированных сервисов:

1. **Orders Service**:

* Управление заказами (создание, просмотр).
* Реализует паттерн **Transactional Outbox**: создание заказа и события о необходимости оплаты происходит в одной
  ACID-транзакции БД.

2. **Payments Service**:

* Управление счетами пользователей (баланс, пополнение).
* Обработка платежей.
* Реализует паттерн **Transactional Inbox**: идемпотентная обработка входящих сообщений (дедупликация).
* Использует **Saga (Choreography)** для асинхронного взаимодействия с сервисом заказов.

3. **Gateway**:

* Единая точка входа для API.
* Агрегатор документации Swagger UI.

4. **Frontend**:

* SPA приложение для взаимодействия с системой (регистрация, пополнение счета, заказы).

## Запуск проекта

Для запуска требуются только **Docker** и **Docker Compose**.

1. Клонируйте репозиторий.
2. Запустите сборку и старт контейнеров:

```bash
docker compose up --build
```
или
```bash
docker-compose up --build
```

(Вам могут понадобиться особые права для запуска этих команд).

*Первый запуск может занять некоторое время из-за компиляции зависимостей.*

## Адреса и Доступы

После успешного запуска приложение доступно по следующим адресам:

| Сервис          | URL                                                                    | Описание                                              |
|:----------------|:-----------------------------------------------------------------------|:------------------------------------------------------|
| **Frontend**    | [http://localhost:3000](http://localhost:3000)                         | Главный интерфейс приложения                          |
| **Gateway API** | [http://localhost:8080](http://localhost:8080)                         | Основной API Endpoint                                 |
| **Swagger UI**  | [http://localhost:8080/swagger-ui/](http://localhost:8080/swagger-ui/) | Объединенная документация всех сервисов               |
| **RabbitMQ UI** | [http://localhost:15672](http://localhost:15672)                       | Панель управления брокером (login: `guest` / `guest`) |

## Сценарий использования

1. Откройте **Frontend** (`http://localhost:3000`).
2. Нажмите **"Register New Account"** (создастся счет с балансом 0).
3. Перейдите во вкладку **"Deposit"** и пополните счет.
4. Перейдите во вкладку **"New Order"** и создайте заказ.
5. В разделе **"My Orders"** статус заказа должен смениться на `Finished` (если средств хватило) или `Cancelled` (если
   нет).
